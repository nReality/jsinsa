High-performance Nodebots with Raspberry Pi

Did you know that you can control a robot with JavaScript and Node.js? If you thought robotics was too hard and that you needed to learn a low level language to get started, think again. Nodebots has been around for a while using the Arduino and libraries like Johnny Five, but it just isn't suitable for some robots. What if you need internet connectivity or to interface directly with computer peripherals? Maybe you want to use computer vision or have other computationally heavy algorithms you need to use?

A better alternative in this case is using the Raspberry Pi, which has all the power you need and can easily interface with other hardware. The downside? Getting Node.js and Johnny Five working on Raspberry Pi requires some effort.

The first part of the talk will touch on how to get everything set up (Which OS you want to install, configuring the Pi for headless access via SSH and getting Node.js, NPM and Johhny Five working). The second part will then focus on developing code for a working robot, simulating the hardware on a laptop and finally seeing it running live on a real robot.

ngResource Scaffolding: A tale of too much work

The presentation is about some of the growing pains we’ve faced in building large, non-trivial Angular applications at Entelect, and how we’ve overcome them. We’ve come up with our own project structures, build processes and design patterns that make our day-to-day development a lot easier.  We’ve also built a new tool to scaffold more boilerplate code for us. It works as part of our build process,  generating Swagger JSON specifications off our backend API, then scaffolding Angular ngResources off that specification. Basically, we think we’ve got a great time saving tools and approaches that we’d like to share.

Event Sourcing: How, Where, Why

Most projects I've worked on have needed a way to see data over time and at any given point in time. I have been involved in architecting solutions like this for many years and only when I learnt about event sourcing did I realise that I am not a unique snowflake with this challenge. The biggest question i come into contact with is "Why would I use an event store?". To answer this question we will explore how and where an event store could work and then take a look at the why you would choose it. We'll see how the GetEventStore steps up to the challenge and where to look to get started. I hope to add a useful tool to your toolbox.

Over the years I thought that to be a good developer I had to be passionate ninja that spends hours and hours researching, learning, and contributing to open source. While at the same time going the working loads of over time and going the extra-mile to make sure my team succeed.

Developing, Fast and Slow

One of the daily challenges we face as developers is staying focussed on the problem we are trying to solve.

A solution came to me from a book by a Nobel Prize winning psychologist Daniel Kahneman: we have to increase our thinking stamina by automating everything that adds cognitive load.

With this talk I will take you through some of his ideas on how the mind works, and then share my views on how they apply to developer workflow. Finally, I will show how to free up our brains to focus on what's important.
